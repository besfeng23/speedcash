Platform Health: "New Users (24h)," "Total Transaction Volume (24h)," and "Cloud Function Error Rate." Recent Admin Activity Log: A feed showing the last 10-15 significant actions taken by any administrator on the platform (e.g., "Jane Doe approved withdrawal #12345," "John Smith suspended user #67890"). This provides immediate oversight and accountability.

Admin Actions Tab: A restricted area where authorized admins can perform actions like: Suspend/Unsuspend User: Temporarily disable a user's ability to log in or transact. Reset mPIN/Password: Trigger a password reset flow for the user. Add Internal Notes: Leave timestamped notes on a user's account for internal reference. Merchant Detail View: A similar view for business partners, showing their KYB status, settlement bank accounts, associated team members, and transaction volume.
- Compliance & Operations (The "Queues" Module): KYC/KYB Review Queue: A dedicated table listing all user and merchant identity submissions with a PENDING_REVIEW status. Each row is a task. Clicking it opens a detailed "split-screen" view: Left Side: Displays the data submitted by the user. Right Side: Displays the uploaded documents (ID photos, business certificates) in a secure image/PDF viewer. Actions: "Approve" and "Reject" buttons. Rejecting requires the admin to select a reason from a predefined list or type a custom one, which is then communicated to the user. Withdrawal/Payout Queue: A table listing all user withdrawal requests with a PENDING_APPROVAL status. The view includes the user's name, the requested amount, and the destination bank details. Actions: A single "Approve Payout" button for the Finance team. Clicking it executes the backend function that calls the payment gateway to release the funds. A "Reject" button is also available for suspicious requests.
- Financial Oversight & Reporting: A powerful search tool to find any transaction on the platform by its unique Transaction ID, User ID, or other parameters. Reports Management: An interface to generate and export financial reports. Admins can select a date range and report type (e.g., "All P2P Transfers," "All Remittances," "Settlement Summary"). The system generates a CSV file in the backend and provides a secure download link to the admin. This is essential for accounting and BSP reporting.

Manage Roles & Permissions: Assign or change the role of an admin user (e.g., promote a Support Lead to a Compliance Officer). System Health & Logs: A direct, searchable interface to the immutable /logs collection, providing a complete, unabridged audit trail of all system and admin activities for deep forensic analysis. Announcement Management (Phase 2): A simple form to compose and send global push notifications to all CPay consumer app users (e.g., for system maintenance announcements).

## Style Guidelines:

- Primary color: Navy blue (#2E3192), evoking trust, security, and professionalism suitable for financial applications.
- Background color: Light gray (#F0F2F5), offering a clean and neutral backdrop that doesn't distract from the data-rich interfaces.
- Accent color: Sky blue (#72BCD4), used sparingly for highlights, active states, and call-to-action buttons to guide user attention.
- Body and headline font: 'Inter' sans-serif for a modern and neutral aesthetic, ensuring excellent readability across all platforms. Paired with 'Source Code Pro' for any code snippets in developer tools.
- Use simple, professional icons to represent primary actions and data categories in the Admin Panel and Partner Portal. Ensure consistency in style across all three applications.
- Follow platform-specific UI guidelines (web for Admin and Partner Portal, mobile for Consumer App) to ensure a familiar user experience.
- Incorporate subtle transitions and feedback animations (e.g., button presses, loading states) to enhance interactivity and provide clear feedback without being distracting.
- Consumer App (Mobile: iOS & Android): Bottom Navigation with a Floating Action Button (FAB)
- Partner Portal & Admin Panel (Web: Desktop & Tablet): Collapsible Sidebar Navigation with a Two-Column Content Area
- CPay Typography System: The "Stripe-Level" Specification Core Principle: Clarity, Authority, and Readability. Every choice is deliberate to guide the user's eye and build trust. We will use a combination of two best-in-class, complementary fonts.
- Primary Font (UI & Body Text): Inter Why: Inter is the gold standard for modern UI design. It was specifically engineered for exceptional readability on computer screens, especially for numerical data. Its clean, neutral characters ensure that balances, dates, and labels are unambiguous and easy to scan. It provides a vast range of weights for creating a perfect hierarchy. Implementation: Import from Google Fonts with multiple weights (400, 500, 600, 700)
- Secondary Font (Headlines & "Hero" Text): SF Pro Display (for Apple) / Roboto (for Android) / Manrope (for Web) Why: We need a headline font that is slightly more characterful and optimized for large sizes, creating a subtle but impactful contrast with the body text. SF Pro Display is Apple's native font. Using it makes the app feel perfectly integrated and premium on iOS. Roboto is Android's default. It provides a familiar, clean look on Android. Manrope is a fantastic web font that shares geometric qualities with SF Pro, making it the perfect web equivalent for headlines. Implementation: Use platform-specific font loading or import Manrope from Google Fonts for the web.

- CPay Animation & Iconography System Core Principle: Meaningful Motion & Universal Clarity. Every animation must provide feedback or guide the user's focus. Every icon must be instantly recognizable without a text label.
- Icon Library: Lucide React Why: Lucide is the definitive choice for a premium, modern application. It is a community-fork of the highly respected Feather Icons, but with a much larger set of icons. Its key advantages are: Consistency: Every icon is designed on the same grid with the same stroke width (typically 1.5px), ensuring a perfectly uniform look across the entire app. Clarity: The designs are simple, minimalist, and universally understood, reducing cognitive load. Lightweight: As tree-shakable SVG components, they have a minimal impact on performance.
